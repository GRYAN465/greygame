# GRAYGAME

一款使用 **Godot 4.6** 制作的 **竖屏 2D 街机式生存/躲避** 游戏。玩家在固定大小的战场内移动，躲避并反击带元素效果的敌人，通过拾取、Buff 和技能管理生命与能量，在持续刷怪与涨难度下追求更高分数。

---

## 运行环境

| 项目 | 说明 |
|------|------|
| 引擎 | Godot **4.6**（项目特性：`4.6`，渲染：`Forward Plus`） |
| 主场景 | 工程内 `main.gd` 对应根节点为 `Main` 的逻辑；若本地缺少主场景 `.tscn` 文件，请在编辑器中**指定或恢复主场景**为包含 `Main` 节点且挂载 `main.gd` 的场景 |
| 窗口 | 默认分辨率 **800×1200**（`canvas_items` 拉伸，适合竖屏） |
| 平台 | 含 `export_presets.cfg`，可按需导出；Windows 下项目配置了 **D3D12** 驱动（见 `project.godot`） |

在 Godot 中打开项目根目录，运行（F5）即可。

## 核心玩法概览

1. **生存与移动**：玩家在屏幕范围内移动，角色使用 `AnimatedSprite2D` 根据朝向切换上下、侧向等动画；受击时播放受击动画。
2. **分数**：`ScoreTimer` 持续为分数 **+1**；击倒普通敌人 **+3**；另有与能量相关的计分（见下「能量与计分」）。
3. **难度随时间增加**：`DifficultyTimer` 会缩短普通怪刷新间隔（有下限），并调整特殊怪刷新节奏，使后期压力更大。
4. **游戏结束**：生命归零后进入 Game Over 流程，停止各类计时器与 BGM，显示结束提示与重新开始入口。

## 玩家能力（`player.gd`）

### 基础属性（可通过编辑器 `@export` 调整）

- **生命**：默认最大 50。  
- **能量**：0～100。  
- **移速**：基础约 310；受状态与技能影响。  
- **攻击力**：基础 4，影响「僚机」每次命中伤害。

### 无敌与受伤

- 受伤后约 **1 秒无敌**（部分伤害可 `ignore_invincible` 无视无敌）。  
- 在特定防御状态下**不再吃异常与部分伤害**（见技能 Q）。

### 元素状态与联动

游戏实现了 **火 / 冰 / 感电 / 潮湿** 四类状态，部分组合会产生质变：

- **燃烧**：持续一段时间，周期扣血。  
- **寒冷**：降速；若**同时潮湿**会 **完全冻结**（移速为 0，有冻结类动画时切换为 `frozen`）。  
- **感电**：周期消耗能量（寒冷时抽能更多）。  
- **潮湿**：周期**回复能量**；与火、感电、冰等有抵消或连携（例如火命中冰会转化为潮湿并返还能量等，具体以脚本逻辑为准）。  
- 部分组合会触发 **爆炸表现**与额外伤害（如电与火/燃烧相关的处理）。

防御状态下（见下）**不会**被加上新的元素效果。

状态会在角色身上用 **VFX 层** 显示不同特效；HUD 上也有对应图标高亮。

### 僚机武器（`attack.tscn` + `attack.gd`）

- 拾取场上 **道具** 可获得环绕玩家的 **Area2D 僚机**，以固定半径**绕角色旋转**，对碰到的敌群单位造成伤害（伤害=当前 `ATK`）。  
- 僚机存在一段时间后由计时器**回收**；拾取新道具会重新发放僚机。  
- **武器转速** 可通过 Buff 提升（`angular_speed` 累加），也可由玩家 `weapon_spin_bonus` 同步增强已有僚机。

### 技能

| 技能 | 能量消耗 | 效果摘要 |
|------|----------|----------|
| **E** | 35 | 以自身为圆心、约 **400** 像素的 **AOE**：范围内敌人受到高额伤害（脚本中为 **999** 一击性质），用于清屏/救急。 |
| **Q** | 20 | 约 **2.5 秒** 进入「防御 / 地城」类状态：免疫/屏蔽元素套用等（`is_inDUN`），并播放对应 VFX。 |
| **R** | 40 | 约 **5 秒** **大幅加速** 移动，**回复生命 10**，带加速线特效。 |
| 蓄能 **J** | 无直接消耗 | **按住** 进入蓄能：移速降低；计时随机增加 **1 / 2 / 5** 点能量；满能量时不能再蓄（避免溢出滥用）。满能量有全局联动效果，见下。 |

### 能量与计分（`main.gd` 中 `_process`）

当玩家能量**达到上限**时，能量会被**清零**，并调用 `beat_mob(10)`：此处实际效果为 **分数 +10**（函数名易误解为“击倒多名敌人”，以代码为准）。  
因此除了击杀与生存计时外，**涨能量并触顶**也会周期性带来额外分数，但会**清空当前能量条**。

## 敌人与特殊机制

### 普通敌人（`mob.gd` / `mob.tscn`）

- 类型**随机**：约 **60% 火、10% 冰、30% 电**（`main.gd` 的 `_pick_mob_type`）。  
- 从**路径**（`MobPath`）上随机点生成，朝场内方向飞入，带随机角度偏移。  
- **RigidBody2D** 运动；冲出屏幕可见区域会被回收。  
- 与**玩家碰撞**时：对玩家造成类型对应的**直接伤害**并**施加元素效果**；玩家**撞击**会对其造成 **1 点**伤害（可配合僚机高攻收割）。  
- 属性（伤害、生命、速度）随类型不同：例如火系偏快高伤，冰系偏肉，电系偏快等（见 `setup_by_type`）。  
- 血条见底先播放受击/死亡动画再销毁，并为主角方的分数 **+3**。

### 特殊敌人（`special_mob.gd` / `special_mob.tscn`）

- 成对生成两只普通怪，从**同一边缘**向场内运动，中间用 **Line2D** 画出**连线**；连线路径上有 **区域判定**。  
- **其他怪物**若进入线形区域会被**立刻清除**（`queue_free`）。  
- **玩家**若与线形区域接触会受到 **10** 点伤害。  
- 任一侧怪物被销毁后父节点会自清理。  

该机制在对抗大量杂兵时改变走位策略，也增加被“线”刮到的风险。

## 拾取与 Buff

### 道具（`item.gd`）

- 周期在随机位置刷新，一段时间未拾取会消失。  
- 效果：**获得/刷新僚机**、**+6 能量**、施加**潮湿**（与全局元素系统联动）。

### 三种 Buff 场景（`buff.gd` / `buff2.gd` / `buff3.gd`）

- **攻击 Buff**：`ATK + 1`（有调试 `print`）。  
- **治疗 Buff**：`heal(4)`。  
- **旋转 Buff**：`add_weapon_spin_speed(0.5)`，提高僚机角速度。  

各 Buff 有存在时间，超时自毁。

## UI（`hud.gd` / `hud.tscn`）

- **分数**、**生命条**、**能量条**。  
- **四元素图标**：根据当前状态显示/隐藏。  
- **开始**按钮与**规则**面板（`RulePanel` 开关）。  
- 开始流程会通过 `Message` 与计时器做简单演出；Game Over 会停止游戏 UI 的显示并切回主菜单式流程。  

> 主界面英文提示仍保留 *“Dodge the Creeps”* 等来自模板的字符串，与当前元素战斗玩法并存，可视作占位文案。

## 组（Groups）与协作

- **`player`**：玩家节点所在组，供 Buff/道具/线形区检测。  
- **`mobs`**：敌人在组中，供僚机、技能 AOE、连线清除等统一遍历。  

技能 E 的 AOE 即通过 `get_tree().get_nodes_in_group("mobs")` 处理范围内单位。

## 已知注意点

- `project.godot` 中主场景为 **UID 引用**；若克隆仓库后无法运行，请检查是否存在对应主场景资源并在编辑器中重新指定 `run/main_scene`。  
- `main.gd` 内 `beat_mob` 在**能量顶满**时只增加**分数**，与函数名字面含义不同，修改逻辑时建议同步命名或加注释。  
- 工程配置里出现 **Jolt 3D 物理** 等条目，本游戏主逻辑为 **2D**；以实际场景树为准。
